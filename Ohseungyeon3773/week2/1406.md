# 연결 리스트
꼬리물기 식으로 저장된 자료구조
배열: 시작점을 표시 + 이후 원소는 공백이 나올 때까지 차례대로 배열됨
연결 리스트: (저장하는 값 + 다음 저장값의 주소)가 한 묶음. 즉, 굳이 서로 붙어있지 않아도 됨.

### 연결 리스트의 성질
- K번째 원소를 확인/변경하기 위해 O(k)가 필요하다
- 임의의 위치에 원소를 추가/임의의 위치에 원소 제거는 O(1)
- 원소들이 메모리 상에 연속해있지 않음 > Cache hit rate가 낮음 BUt 할당은 쉬움

### 연결 리스트의 종류
- 단일 연결 리스트
다음 원소의 위치만 같이 저장됨

- 이중 연결 리스트
다음 원소의 위치 + 이전 원소의 위치가 같이 저장됨

- 원형 연결 리스트
끝이 처음과 연결된 리스트
단일 연결 리스트/이중 연결 리스트 중 끝이 처음과 연결된 리스트는 모두 원형 연결 리스트에 해당됨

### 배결과 연결 리스트의 차이
직선형 구조 <> 꼬리물기 구조로 생기는 차이들
배열, 리스트 순으로 쓰여짐

- K번째 원소의 접근
O(1) <> O(k)

- 임의 위치에 원소 추가/제거
O(N) <> O(1) (단, 위치를 알고 있다는 전제)

- 메모리 상의 배치
연속 <> 불연속

- 추가적으로 필요한 공간(Overhead)
X <> O(N)
