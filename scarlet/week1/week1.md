## Greedy algorithm
---

`매 순간 가장 최적인 답을 근시안적으로 택하는 알고리즘`
`관찰을 통해 탐색 범위를 줄여나가는 알고리즘`

`예시) Merge sort : O(N+M)`

`→ 일단 해보고 틀리면 빠르게 손절. 그리디 알고리즘 유형으로 착각하지 않게 주의!`

    `정당성 검토 과정 필요.`

### `해결하는 방법`

1. `선택 절차 (Selection Procedure): 현재 상태에서의 최적의 해답 선택.`
2. `적절성 검사 (Feasibility Check): 선택된 해가 문제의 조건을 만족하는지 검사.`
3. `해답 검사 (Solution Check): 원래의 문제가 해결되었는지 검사하고, 해결되지 않았다면 선택 절차로 돌아가 위의 과정을 반복.`

### `적용시 성립되어야 할 조건`

`그리디 알고리즘을 적용하는 경우 2개의 조건이 성립되어야만 한다.`

1. `탐욕적 선택 속성 (Greedy Choice Property) : 앞의 선택이 이후의 선택에 영향을 주지 않는다.`
2. `최적 부분 구조 (Optimal Substructure) : 문제에 대한 최종 해결 방법은 부분 문제에 대한 최적 문제 해결 방법으로 구성된다.`

`이러한 조건이 성립하지 않는 경우에는 탐욕 알고리즘은 최적해를 구하지 못한다. 
하지만, 이러한 경우에도 탐욕 알고리즘은 근사 알고리즘으로 사용이 가능할 수 있으며, 대부분의 경우 계산 속도가 빠르기 때문에 실용적으로 사용할 수 있다.`

**`그리디 알고리즘은 항상 최적의 결과를 도출하는 것은 아니지만, 어느 정도 최적에 근사한 값을 빠르게 도출할 수 있는 장점이 있다. → 그리디 알고리즘은 근사 알고리즘으로 사용할 수 있다.`**

**`문제 예시`**

`11047(동전 0), 1931(회의실 배정), 2217(로프), 1026(보물)`
